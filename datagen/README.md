# Parallel Data Generator for CloudEdgeBench

### Config
Edit `config.yml`


Supported patterns:
- uniform
- poisson
- zipfian
- burst
  - fraction
  - distribution

Supported data sources:
- `csv`
- `image_folder`

`image_folder` scans the configured directory recursively for `.png` files and publishes each frame as CSV in this format:
`id,timestamp,width,height,data`

For image frames:
- `id` is the file stem
- `timestamp` is generated by the existing pattern scheduler
- `width` and `height` come from the PNG header
- `data` is the raw PNG file bytes encoded as `base64` or `hex`

### Patterns
If you would like to observe how generated data looks like, use the `patternTest.py` script. It generates `series.png` to visualize the generated data.
```
python3 patternTest_timeseries.py --host 127.0.0.1 --port 8080 --duration 10 --out series.png
```

### Run
Run the executable `datagen`, which reads the configuration from `config.yml`.

### Docker Deployment
The repository includes a self-contained Docker setup under [`docker/`](./docker).

Build and start the default TCP-based setup from the repository root:
```bash
docker compose -f docker/compose.yaml up --build
```

This will:
- build the `datagen` image
- mount `data/` into the container at `/app/data`
- mount [`docker/config.yaml`](./docker/config.yaml) as the active config
- publish ports `8080`, `8081`, and `8082`

To start the optional Mosquitto broker as well, use the MQTT profile and the MQTT-enabled config:
```bash
DATAGEN_CONFIG=/app/config/config.mqtt.yaml docker compose -f docker/compose.yaml --profile mqtt up --build
```

The Docker-specific configuration files are:
- [`docker/config.yaml`](./docker/config.yaml) for socket output only
- [`docker/config.mqtt.yaml`](./docker/config.mqtt.yaml) for socket output plus MQTT

For more detail, see [`docker/README.md`](./docker/README.md).
